# Glyph

> **代码即艺术。让美被看见。**
>
> *Code is art. Glyph makes it visible.*

---

Glyph 将源代码转化为**可以挂上墙的抽象画**。它解析代码的 AST（抽象语法树），提取其中的美学特征——结构、嵌套、流动、留白——然后将它们映射为具有审美意图的视觉作品。

这不是代码高亮，不是可视化工具。这是一场**让程序员感受到的代码之美，穿透屏幕，被任何人看见**的实验。

---

## 🎨 艺术哲学

### 代码中蕴含四种美

| 美的类型 | 代码中的体现 | 艺术对应 |
|----------|-------------|----------|
| **结构之美** | 函数的嵌套、作用域的分层、模块的边界 | 蒙德里安的构成主义 — 矩形色块分割 |
| **流动之美** | 管道链、递归调用、数据在函数间流转 | 波洛克的行动绘画 — 笔触的方向与轨迹 |
| **呼吸之美** | 空行、注释、恰当的空白间距 | 约翰·凯奇的沉默美学 — 留白即表达 |
| **文字之美** | 变量命名形成的语义场 | 具体诗 (Concrete Poetry) — 文字即视觉 |

### 我们的映射规则

```
代码属性           →  视觉元素           艺术参考
─────────────────────────────────────────────────────
函数边界           →  色块区域            蒙德里安、马列维奇
嵌套深度           →  色彩饱和度          约瑟夫·阿尔伯斯色彩理论
函数调用关系        →  连接线网络          索尔·勒维特墙绘
代码行长度          →  笔触宽度            弗朗兹·克莱恩书法抽象
空白与注释          →  画面负空间（留白）   东方留白美学
变量作用域          →  色彩渐变/晕染       马克·罗斯科色域绘画
代码复杂度          →  纹理密度            艾格尼丝·马丁的网格
```

## 🖼️ 预期效果

一段精心编写的递归函数，经由 Glyph 渲染后：

> 画面中央一块浓郁的靛蓝色矩形（函数本身），内部嵌套着渐浅的蓝色方块（子调用），方块之间有细密的金色连线（调用关系），四周是大面积的留白（函数的呼吸空间）。整幅画遵循黄金比例构图，色彩来自精心挑选的调色板——这不是随机的配色，而是**有审美意图的艺术创作**。

## 🏛️ 艺术风格系统

不同的代码范式产生不同的画风：

| 代码风格 | 画风 | 视觉特征 |
|----------|------|----------|
| **函数式** | 冷抽象 / 几何构成 | 蒙德里安模式 — 清晰边界、纯色块 |
| **面向对象** | 暖色调 / 有机形态 | 康定斯基模式 — 交织、互动、暖色 |
| **脚本/胶水** | 行动绘画 | 波洛克模式 — 泼溅、流动、能量 |
| **库/框架** | 极简主义 | 禅意模式 — 大面积留白、精确结构 |

## 🗺️ 路线图

- [ ] **Phase 1: 解析层** — AST 提取器，支持多语言（TypeScript/Python/Rust/Go），输出"美学特征向量"
- [ ] **Phase 2: 渲染引擎** — 构图算法 + SVG 渲染器，将特征映射为视觉元素
- [ ] **Phase 3: 风格系统** — 实现四种核心画风，支持自动识别代码范式
- [ ] **Phase 4: 交互与导出** — Web 界面、高清 PNG 导出、打印规格适配

## 🔬 同类项目调研

调研了 7 个相关开源项目后，我们发现了一个清晰的市场空白：

```
   简单映射                            AI 黑盒
   ────────                           ────────
   │                                       │
   │  code2art    sudorandom/art           │  CodeMorpheus
   │  (字节→像素)  (quine实验)              │  (一键AI自画像)
   │                                       │  Syntaxesia
   │              ⬆ GLYPH                 │  (Gemini 生成)
   │            在这里                     │
   │                                       │  GitAura
   │    • AST 结构分析（确定性 + 可解释）    │  (3D AI 场景)
   │    • 艺术理论支撑的映射规则            │
   │    • 每个视觉元素可追溯到代码          │
   └───────────────────────────────────────┘
```

所有现有项目都落在光谱两端——要么太简单（无语义理解），要么太黑盒（AI 生成，不可控）。**中间地带完全空白。**

> 📄 完整调研报告：[docs/research.md](docs/research.md)

**Glyph 的护城河**：
- ✅ **确定性 + 可解释** — 基于 AST，同样的代码永远生成同样的画，每个像素都有来历
- ✅ **有艺术理论支撑** — 映射规则对应具体的艺术流派，不是拍脑袋的配色
- ✅ **结构驱动，非 AI 幻觉** — 画的是代码本身的结构，不是 AI "觉得"代码长什么样
